## 1. 개요

온라인 스토어 프로모션용 크롬 다이노 게임. 커스텀 캐릭터(쥐)로 교체 가능하고, 유저별 점수를 저장하여 상위 N명 랭킹을 관리한다. iframe으로 삽입되므로 완전히 독립적으로 동작해야 한다.

| 항목 | 현재 | 목표 |
| ---- | ---- | ---- |
| 캐릭터 | 고정된 픽셀아트 | 이미지 파일 교체 가능 |
| 장애물 | 고정된 픽셀아트 | 이미지 파일 교체 가능 |
| 인트로 화면 | 없음 (바로 시작) | 타이틀 표시 후 시작 |
| 점수 저장 | 로컬 개발 환경만 | 프로덕션 배포, 상위 N개만 보관, 중복 방지 |
| 랭킹 UI | 게임 화면에 랭킹 노출 | 유저에게 미노출, 어드민 페이지에서 별도 관리 |
| 유저 정보 | 이름만 저장 | 이름 + 연락처 + 접속 정보 저장 |
| 배포 | 프론트만 배포 | 프론트 + 백엔드 모두 프로덕션 배포 |
| iframe | 미지원 | iframe 삽입 최적화 |

## 2. 화면 설계

### 2.0 인트로 화면

```
┌─────────────────────────────────────────────────┐
│                                                 │
│                                                 │
│             Hellodude                           │
│           Running Mouse                         │
│                                                 │
│                                                 │
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 이미지 준비 완료 후

┌─────────────────────────────────────────────────┐
│                                                 │
│                                                 │
│             Hellodude                           │
│           Running Mouse                         │
│                                                 │
│         PRESS SPACE TO START                    │
│  ─────────────────────────────────────────────── │
│           or tap the screen                     │
└─────────────────────────────────────────────────┘

        ↓ Space/Tap 입력 시

┌─────────────────────────────────────────────────┐
│                                                 │
│                    ↓ (타이틀 아래로 떨어짐)       │
│             Hellodude                           │
│           Running Mouse                         │
│  🐭→                                            │  ← 쥐가 좌측에서 달려나옴
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 타이틀 "Hellodude" / "Running Mouse"를 화면 중앙에 표시
- 모든 이미지 준비 완료 → "PRESS SPACE TO START" 텍스트 표시
- Space/Tap → 타이틀이 아래로 떨어지는 애니메이션 + 캐릭터(쥐)가 좌측에서 달려나오며 게임 시작

### 2.1 게임 화면

```
┌─────────────────────────────────────────────────┐
│  HI 00350    00127                              │  ← 점수 (우상단)
│                                                 │
│                                                 │
│                                                 │
│        🐭          🌵       🌵🌵    🦅          │  ← 캐릭터 + 장애물
│  ─────────────────────────────────────────────── │  ← 지면
│                                                 │
└─────────────────────────────────────────────────┘
```

- 랭킹, 이름 입력란 없음 — 게임 화면만 표시
- 반응형: 내부 해상도 고정, 화면 크기에 맞춰 자동 스케일링

### 2.2 Game Over 상태

```
┌─────────────────────────────────────────────────┐
│  HI 00350    00127                              │
│                                                 │
│            G A M E  O V E R                     │
│                                                 │
│        🐭          🌵                           │
│  ─────────────────────────────────────────────── │
│                                                 │
│         [다시하기]    [순위작성]                   │  ← 버튼
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ '순위작성' 클릭 시

┌─────────────────────────────────────────────────┐
│  HI 00350    00127                              │
│                                                 │
│            G A M E  O V E R                     │
│                                                 │
│        🐭          🌵                           │
│  ─────────────────────────────────────────────── │
│                                                 │
│   이름:    [____________]                        │
│   연락처:  [____________]                        │
│   점수:    127                                   │  ← 자동 표시 (읽기 전용)
│                                                 │
│         [제출]      [취소]                        │
│                                                 │
└─────────────────────────────────────────────────┘
```

- '다시하기' → 게임 즉시 재시작
- '순위작성' → 이름/연락처 입력 폼 표시
- '제출' → 점수 저장 (이름, 연락처, 점수 전송 / 접속 정보는 서버에서 자동 저장)
- '취소' → 입력 폼 닫기, 다시하기/순위작성 버튼 복귀

## 3. 화면 흐름도

```
┌────────┐  이미지 준비 완료  ┌────────┐  Space/Tap  ┌──────────┐  충돌  ┌───────────┐
│ 로딩   │──────────────────▶│ 인트로 │────────────▶│ 게임 중  │──────▶│ 게임오버  │
└────────┘                   └────────┘             └──────────┘       └───────────┘
                                                        ▲                    │
                                                        │  '다시하기' 클릭    │
                                                        └────────────────────┘
                                                        ▲                    │
                                                        │  '순위작성' → 제출/취소 후 '다시하기'
                                                        └────────────────────┘
```

- **로딩**: 페이지 열림 → 이미지 준비 중. 타이틀만 표시.
- **인트로**: 이미지 준비 완료 → "PRESS SPACE TO START" 표시
- **게임 중**: Space/Tap → 타이틀 떨어지는 연출 + 캐릭터 등장 → 게임 진행
- **게임오버**: 충돌 → GAME OVER + '다시하기'/'순위작성' 버튼 표시

## 4. 주요 기능

### 4.1 커스텀 스프라이트 (이미지 교체 시스템)

이미지 파일 교체만으로 캐릭터와 장애물을 변경할 수 있는 시스템. 프로모션마다 재사용 가능.

**이미지 포맷**: WebP 권장 (PNG도 허용)

**모든 이미지는 2프레임 루프로 동작** — 달리기, 숙이기, 날아다니는 장애물 모두 2장 교대.

**각 아이템별 권장 이미지 크기**:

| 아이템 | 파일명 | 크기 (px) | 비고 |
| ------ | ------ | --------- | ---- |
| 캐릭터 달리기 | dino-run-1.webp, dino-run-2.webp | 88 x 96 | 2프레임 루프 |
| 캐릭터 숙이기 | dino-duck-1.webp, dino-duck-2.webp | 118 x 60 | 2프레임 루프, 넓고 낮게 |
| 캐릭터 점프 | dino-jump-1.webp, dino-jump-2.webp | 88 x 96 | 2프레임 루프 |
| 캐릭터 사망 | dino-dead-1.webp, dino-dead-2.webp | 88 x 96 | 2프레임 루프 |
| 장애물 소 | obstacle-sm-1.webp, obstacle-sm-2.webp | 34 x 70 | 지면 장애물 |
| 장애물 대 | obstacle-lg-1.webp, obstacle-lg-2.webp | 50 x 96 | 지면 장애물 |
| 장애물 더블 | obstacle-db-1.webp, obstacle-db-2.webp | 80 x 96 | 지면 장애물 |
| 날아다니는 장애물 | obstacle-fly-1.webp, obstacle-fly-2.webp | 84 x 60 | 공중 장애물 |
| 지면 타일 | ground.webp | 2400 x 24 | 가로로 반복 스크롤 |

> 크기는 게임 내부 기준. 실제 화면에서는 자동 스케일링 적용.
> 초기 구현 시 이미지 리소스 없이 컬러 박스 placeholder로 동작.

**이미지 교체 방법**: `assets/sprites/` 폴더의 이미지 파일만 교체 + 설정 파일에서 크기 조정.

### 4.2 점수 시스템

**저장 정보**: 이름, 연락처, 점수, 접속 정보(자동), 등록 시각(자동)

**중복 방지**: 연락처를 유저의 고유 식별자로 사용. 동일 연락처의 유저는 **최고 점수만 유지**.

**세션 저장**: 유저가 한 번 이름/연락처를 입력하면 브라우저에 저장. 이후 게임오버 시 '순위작성' 클릭하면 이전 입력값이 자동으로 채워짐. 유저는 수정 가능.

**연락처 검증**:
- 전화번호 형식: `010-XXXX-XXXX` (하이픈 포함)
- 입력 시 실시간 포맷팅 + 제출 전 검증
- → **연락처 형식은 구현 시점에 최종 결정** (전화번호 vs 이메일 vs 기타)

**상위 N개만 보관**: 기본값 100건. 환경 설정으로 조정 가능.

### 4.3 어드민 페이지

별도 페이지에서 랭킹을 관리.

```
┌─────────────────────────────────────────────────┐
│              DINO GAME ADMIN                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  [로그인 폼]                                     │
│  비밀번호: [____________]  [로그인]               │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 로그인 성공 후

┌─────────────────────────────────────────────────┐
│  DINO GAME ADMIN                    [로그아웃]   │
├─────────────────────────────────────────────────┤
│                                                 │
│  RANKING (전체 N건)                              │
│                                                 │
│  #  | 이름    | 연락처       | 점수 | 날짜       │
│  1  | alice   | 010-1234-... | 1200 | 03-30     │
│  2  | bob     | 010-5678-... |  980 | 03-29     │
│  ...                                            │
│  100 | ...                                      │
│                                                 │
│  각 행: [수정] [삭제]                             │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 비밀번호 기반 로그인
- 전체 랭킹 조회, 수정, 삭제 가능

### 4.4 iframe 삽입

스토어 페이지에 iframe으로 삽입:

```html
<iframe
  src="https://your-domain.com/dino-game.html"
  width="100%"
  height="300"
  frameborder="0"
  allow="autoplay"
  style="max-width: 640px;"
></iframe>
```

- 보안 설정으로 허용된 도메인에서만 삽입 가능
- iframe 내 클릭 시 자동 포커스 처리

## 5. 설계 결정 근거

**왜 이미지 교체 시스템인가?**
- 현재 캐릭터는 코드에 고정되어 있어 교체가 사실상 불가능
- 이미지 파일 교체만으로 디자이너가 직접 캐릭터를 바꿀 수 있어야 프로모션마다 재사용 가능

**왜 인트로 화면을 추가하는가?**
- 이미지 로딩 시간 동안 빈 화면 대신 브랜드 노출 (Hellodude)
- 이미지 로드 완료 전 게임 시작을 방지하여 깨짐 방지

**왜 유저 화면에서 랭킹을 숨기는가?**
- 프로모션 목적상 순위 경쟁보다 참여 유도가 중요
- 연락처 등 개인정보가 포함되므로 유저에게 직접 노출 부적절
- 어드민이 별도로 관리하여 경품 지급 등에 활용

**왜 유저당 최고 점수만 유지하는가?**
- 같은 유저가 100번 플레이하면 랭킹을 독점하는 문제 방지
- 연락처로 고유 식별 — 이름은 변경 가능하지만 연락처는 고유

## 7. 인프라 비용 비교

| 옵션 | 백엔드 | DB | 월 비용 |
| ---- | ------ | -- | ------- |
| Railway | $5 크레딧/월 | PostgreSQL 500MB | $0~5 |
| Render | 무료 (cold start 있음) | PostgreSQL (무료 90일) | $0 |
| Fly.io | 무료 (3 shared VMs) | PostgreSQL 1GB 무료 | $0 |

> 프론트엔드는 모든 옵션에서 GitHub Pages (무료).
> 구현 시점에 트래픽 예측과 운영 편의성에 따라 선택.

## 8. 구현 마일스톤

| Step | 목표 | 주요 산출물 |
| ---- | ---- | ---------- |
| Step 1 | 인트로 화면 + 이미지 교체 시스템 | 인트로 연출, 이미지 로딩, placeholder 동작 |
| Step 2 | 게임오버 UX + 순위작성 | 다시하기/순위작성 버튼, 입력 폼, 연락처 검증 |
| Step 3 | 백엔드 개선 | 점수 중복 방지, 어드민 인증, 유저 정보 확장 |
| Step 4 | 어드민 페이지 | 로그인, 랭킹 조회/수정/삭제 |
| Step 5 | iframe 최적화 + 배포 | iframe 보안, 프로덕션 배포, 테스트 |

> 각 Step 완료 후 리뷰 진행, 확인 후 다음 Step 착수.

## 9. 열린 결정 사항

| 항목 | 옵션 | 결정 시점 |
| ---- | ---- | --------- |
| 배포 플랫폼 | Railway vs Fly.io vs Render | Step 5 시작 전 |
| 랭킹 보관 수 (N) | 50 / 100 / 200 | Step 3 |
| 어드민 인증 방식 | 단순 API Key vs JWT vs 세션 | Step 3 |
| 프로모션 종료 후 데이터 처리 | 삭제 / 아카이브 / 유지 | 프로모션 기획 확정 시 |
| 부모 페이지와 통신 | 필요 여부는 스토어 측 요구사항에 따라 | Step 5 |
| 연락처 형식 | 전화번호 (`010-XXXX-XXXX`) / 이메일 / 기타 | Step 2 |
