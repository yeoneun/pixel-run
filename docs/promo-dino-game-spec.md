## 1. 개요

온라인 스토어 프로모션용 크롬 다이노 게임. 커스텀 캐릭터(쥐)로 교체 가능하고, 유저별 점수를 저장하여 상위 N명 랭킹을 관리한다. iframe으로 삽입되므로 완전히 독립적으로 동작해야 한다.

| 항목 | 현재 | 목표 |
| ---- | ---- | ---- |
| 캐릭터 | 고정된 픽셀아트 | 이미지 파일 교체 가능 |
| 장애물 | 고정된 픽셀아트 | 이미지 파일 교체 가능 |
| 인트로 화면 | 없음 (바로 시작) | 타이틀 표시 후 시작 |
| 점수 저장 | 로컬 개발 환경만 | 프로덕션 배포, 상위 N개만 보관, 중복 방지 |
| 랭킹 UI | 게임 화면에 랭킹 노출 | 유저에게 미노출, 어드민 페이지에서 별도 관리 |
| 유저 정보 | 이름만 저장 | 연락처 + 접속 정보 저장 |
| 배포 | 프론트만 배포 | 프론트 + 백엔드 모두 프로덕션 배포 |
| iframe | 미지원 | iframe 삽입 최적화 |

## 2. 화면 설계

### 2.0 인트로 화면

```
┌─────────────────────────────────────────────────┐
│                                                 │
│                                                 │
│             Hellodude                           │
│           Running Mouse                         │
│                                                 │
│                                                 │
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 이미지 준비 완료 후

┌─────────────────────────────────────────────────┐
│                                                 │
│                                                 │
│             Hellodude                           │
│           Running Mouse                         │
│                                                 │
│         PRESS SPACE TO START                    │
│  ─────────────────────────────────────────────── │
│           or tap the screen                     │
└─────────────────────────────────────────────────┘

        ↓ Space/Tap 입력 시

┌─────────────────────────────────────────────────┐
│                                                 │
│                    ↓ (타이틀 아래로 떨어짐)       │
│             Hellodude                           │
│           Running Mouse                         │
│  🐭→                                            │  ← 쥐가 좌측에서 달려나옴
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 타이틀 "Hellodude" / "Running Mouse"를 화면 중앙에 표시
- 모든 이미지 준비 완료 → "PRESS SPACE TO START" 텍스트 표시
- Space/Tap → 타이틀이 아래로 떨어지는 애니메이션 + 캐릭터(쥐)가 좌측에서 달려나오며 게임 시작

### 2.1 게임 화면

```
┌─────────────────────────────────────────────────┐
│                               00127              │  ← 점수 (우상단)
│                                                 │
│                                                 │
│                                                 │
│        🐭          🌵       🌵🌵    🦅          │  ← 캐릭터 + 장애물
│  ─────────────────────────────────────────────── │  ← 지면
│                                                 │
└─────────────────────────────────────────────────┘
```

- 게임 화면만 표시 (랭킹, 입력란 없음)
- 반응형: 내부 해상도 고정, 화면 크기에 맞춰 자동 스케일링
- **속도 증가**: 게임이 진행될수록 (점수가 올라갈수록) 화면 스크롤 속도가 점점 빨라져 난이도가 상승합니다

### 2.2 Game Over 상태

```
┌─────────────────────────────────────────────────┐
│                               00127              │
│                                                 │
│            G A M E  O V E R                     │
│                                                 │
│        🐭          🌵                           │
│  ─────────────────────────────────────────────── │
│                                                 │
│         [다시하기]    [순위작성]                   │  ← 버튼
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ '순위작성' 클릭 시

┌─────────────────────────────────────────────────┐
│                               00127              │
│                                                 │
│            G A M E  O V E R                     │
│                                                 │
│        🐭          🌵                           │
│  ─────────────────────────────────────────────── │
│                                                 │
│   연락처:  [____________]                        │
│   점수:    127                                   │  ← 자동 표시 (읽기 전용)
│                                                 │
│         [제출]      [취소]                        │
│                                                 │
└─────────────────────────────────────────────────┘
```

- '다시하기' → 게임 즉시 재시작
- '순위작성' → 연락처 입력 폼 표시 (개인정보 수집 동의 체크박스 포함)
- '제출' → 점수 저장 (연락처, 점수 전송 / 접속 정보는 서버에서 자동 저장)
  - 최고점 갱신 시 → "점수가 저장되었습니다!"
  - 이미 더 높은 점수가 있을 시 → "이미 더 높은 점수(500점)가 기록되어 있습니다. 더 높은 점수를 달성하면 갱신됩니다!"
- '취소' → 입력 폼 닫기, 다시하기/순위작성 버튼 복귀

## 3. 화면 흐름도

```
┌────────┐  이미지 준비 완료  ┌────────┐  Space/Tap  ┌──────────┐  충돌  ┌───────────┐
│ 로딩   │──────────────────▶│ 인트로 │────────────▶│ 게임 중  │──────▶│ 게임오버  │
└────────┘                   └────────┘             └──────────┘       └───────────┘
                                                        ▲                    │
                                                        │  '다시하기' 클릭    │
                                                        └────────────────────┘
                                                        ▲                    │
                                                        │  '순위작성' → 제출/취소 후 '다시하기'
                                                        └────────────────────┘
```

- **로딩**: 페이지 열림 → 이미지 준비 중. 타이틀만 표시.
- **인트로**: 이미지 준비 완료 → "PRESS SPACE TO START" 표시
- **게임 중**: Space/Tap → 타이틀 떨어지는 연출 + 캐릭터 등장 → 게임 진행
- **게임오버**: 충돌 → GAME OVER + '다시하기'/'순위작성' 버튼 표시

## 4. 주요 기능

### 4.1 커스텀 스프라이트 (이미지 교체 시스템)

**이미지 포맷**: WebP 권장 (PNG도 허용)

**모든 이미지는 2프레임 루프로 동작** — 달리기, 숙이기, 날아다니는 장애물 모두 2장 교대.

**각 아이템별 권장 이미지 크기**:

| 아이템 | 파일명 | 게임 내 크기 (px) | 권장 원본 크기 (px) | 비고 |
| ------ | ------ | ----------------- | ------------------- | ---- |
| 캐릭터 달리기 | dino-run-1.webp, dino-run-2.webp | 88 x 96 | 176 x 192 (2배) | 2프레임 루프 |
| 캐릭터 숙이기 | dino-duck-1.webp, dino-duck-2.webp | 118 x 60 | 236 x 120 (2배) | 2프레임 루프, 넓고 낮게 |
| 캐릭터 점프 | dino-jump-1.webp, dino-jump-2.webp | 88 x 96 | 176 x 192 (2배) | 2프레임 루프 |
| 캐릭터 사망 | dino-dead-1.webp, dino-dead-2.webp | 88 x 96 | 176 x 192 (2배) | 2프레임 루프 |
| 장애물 소 | obstacle-sm-1.webp, obstacle-sm-2.webp | 34 x 70 | 68 x 140 (2배) | 지면 장애물 |
| 장애물 대 | obstacle-lg-1.webp, obstacle-lg-2.webp | 50 x 96 | 100 x 192 (2배) | 지면 장애물 |
| 장애물 더블 | obstacle-db-1.webp, obstacle-db-2.webp | 80 x 96 | 160 x 192 (2배) | 지면 장애물 |
| 날아다니는 장애물 | obstacle-fly-1.webp, obstacle-fly-2.webp | 84 x 60 | 168 x 120 (2배) | 공중 장애물 |
| 지면 타일 | ground.webp | 2400 x 24 | 4800 x 48 (2배) | 가로로 반복 스크롤 |

> **게임 내 크기**는 게임 화면에서 실제로 표시되는 크기입니다. **권장 원본 크기**는 고해상도 화면에서도 선명하게 보이도록 2배 크기로 제작한 원본입니다. 원본 이미지를 전달해 주시면, 게임에서 자동으로 적절한 크기로 축소하여 사용합니다.
> 초기 구현 시 이미지 리소스 없이 컬러 박스 placeholder로 동작합니다.

**이미지 교체 방법**: `assets/sprites/` 폴더의 이미지 파일만 교체 + 설정 파일에서 크기 조정.

### 4.2 점수 시스템

**저장 정보**: 연락처, 점수, 접속 정보(자동), 등록 시각(자동)

**접속 정보** (서버에서 자동 수집):
| 항목 | 설명 | 예시 |
| ---- | ---- | ---- |
| IP 주소 | 유저의 인터넷 접속 주소 | `203.0.113.45` |
| User-Agent | 유저의 브라우저/기기 정보 | `Chrome 120 / iPhone` |
| Referrer | 어느 페이지에서 게임에 접속했는지 | `store.example.com/promo` |

> 접속 정보는 동일 연락처로 여러 기기에서 접속하는 경우 등을 구분하기 위해 수집합니다.

**개인정보 수집 동의**: 순위작성 폼에 개인정보 수집 동의 영역을 포함합니다. 동의하지 않으면 제출할 수 없습니다.

동의 화면에 포함해야 할 필수 고지 항목 (개인정보보호법 제15조, 제22조):
| 항목 | 내용 |
| ---- | ---- |
| 수집·이용 목적 | 프로모션 순위 관리 및 경품 지급 |
| 수집 항목 | 전화번호, IP 주소, 브라우저/기기 정보, 접속 경로 |
| 보유 및 이용 기간 | 프로모션 종료 후 30일 이내 파기 (구체적 기간은 확정 필요) |
| 동의 거부 권리 및 불이익 | 동의를 거부할 수 있으며, 거부 시 순위 등록이 불가합니다 |

형식 요구사항:
- 동의 내용은 다른 화면 요소와 명확히 구분되어야 합니다

> 최종 약관 문구는 클라이언트 측에서 제공해야 합니다

**중복 방지**: 같은 연락처(전화번호)로는 **가장 높은 점수 1건만** 저장됩니다. 예를 들어 이전에 500점을 기록한 유저가 다시 플레이하여 300점에서 '순위작성'을 눌러도, 이미 저장된 500점보다 낮으므로 점수가 갱신되지 않습니다. 반대로 700점을 기록하면 기존 500점이 700점으로 갱신됩니다.

**연락처 검증**:
- 전화번호 형식: `010-XXXX-XXXX` (하이픈 포함)
- 입력 시 숫자만 입력하면 자동으로 하이픈이 추가됩니다 (예: `01012345678` → `010-1234-5678`)
- 올바른 전화번호 형식이 아니면 제출할 수 없습니다

**상위 N개만 보관**: 점수가 높은 순서대로 최대 500건까지만 저장됩니다. 500건이 이미 차 있는 상태에서 새로운 점수가 들어오면, 기존 500등보다 높은 점수만 저장되고 가장 낮은 점수가 삭제됩니다.

**점수 조작 방지**: 경품이 걸린 프로모션이므로, 점수를 임의로 조작하여 제출하는 것을 방지해야 합니다. 서버에서 점수 제출 시 실제 플레이 시간을 함께 검증합니다. 예를 들어 게임 시작 후 3초 만에 10,000점을 제출하는 것은 정상적인 플레이로 볼 수 없으므로 거부됩니다.

### 4.3 어드민 페이지

별도 페이지에서 랭킹을 관리.

```
┌─────────────────────────────────────────────────┐
│              DINO GAME ADMIN                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  [로그인 폼]                                     │
│  비밀번호: [____________]  [로그인]               │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 로그인 성공 후

┌─────────────────────────────────────────────────┐
│  DINO GAME ADMIN                    [로그아웃]   │
├─────────────────────────────────────────────────┤
│                                                 │
│  RANKING (전체 N건)                              │
│                                                 │
│  #  | 연락처          | 점수 | 날짜       │
│  1  | 010-1234-5678  | 1200 | 03-30     │
│  2  | 010-5678-1234  |  980 | 03-29     │
│  ...                                            │
│  100 | ...                                      │
│                                                 │
│  각 행: [수정] [삭제]                             │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 비밀번호 기반 로그인
- 로그인 시도 횟수 제한 (5회 실패 시 일시 차단)
- 일정 시간 동안 조작이 없으면 자동 로그아웃
- 연락처는 목록에서 일부가 가려진 상태로 표시 (예: `010-****-5678`). 전체 번호는 개별 항목 클릭 시에만 확인 가능
- 전체 랭킹 조회, 수정, 삭제 가능
- HTTPS 필수 적용

### 4.4 스토어 페이지 삽입

이 게임은 독립된 웹페이지로 만들어지며, 온라인 스토어 페이지 안에 **작은 창(iframe)** 형태로 끼워 넣을 수 있습니다. 스토어 페이지를 방문한 고객은 별도의 페이지 이동 없이, 해당 위치에서 바로 게임을 플레이할 수 있습니다.

- 게임 영역을 클릭하면 자동으로 게임에 포커스가 맞춰져 바로 조작할 수 있습니다
- 허용된 스토어 페이지에서만 게임이 동작하도록 보안 설정이 적용됩니다

## 5. FAQ

**왜 유저당 최고 점수만 유지하나요?**
- 같은 유저가 여러 번 플레이하면 랭킹을 독점하는 문제를 방지하기 위해, 한 유저(같은 전화번호)당 가장 높은 점수 1건만 저장합니다.

**왜 상위 500건만 보관하나요?**
- 프로모션 기간 동안 수천 건 이상의 데이터가 쌓일 수 있으나, 경품 지급 등 실질적으로 필요한 데이터는 상위권에 한정됩니다. 불필요하게 많은 데이터를 보관하지 않고 상위 500건만 유지하여 관리를 간소화합니다.

**개인정보 수집 동의 절차가 왜 필요한가요?**
- 전화번호는 한국 개인정보보호법상 명백한 개인정보입니다. 유저가 직접 입력했더라도 그것만으로는 법적 "동의"로 인정되지 않으며, 수집 목적·항목·보관 기간을 고지한 뒤 명시적 동의(체크박스 등)를 받아야 합니다.
- 동의 절차 없이 수집할 경우 과태료(최대 5,000만 원) 부과 대상이 될 수 있고, 유저가 개인정보보호위원회에 신고하면 조사 대상이 됩니다.
- 또한 경품 지급 시 당첨자가 "동의한 적 없다"고 이의를 제기할 수 있어, 프로모션 운영에도 지장이 생길 수 있습니다.

```
       ├─ '제출'
       │    ├─ 최고점 갱신 → "점수가 저장되었습니다!" → 버튼 복귀
       │    └─ 기존 점수가 더 높음 → "이미 더 높은 점수가 기록되어 있습니다" → 버튼 복귀
       │
       └─ '취소' → 버튼 복귀
```
