## 1. 개요

온라인 스토어 프로모션용 크롬 다이노 게임. 커스텀 캐릭터(쥐)로 교체 가능하고, 일정 점수 도달 시 해피엔딩 연출이 재생된다. iframe으로 삽입되므로 완전히 독립적으로 동작해야 한다.

| 항목 | 현재 | 목표 |
| ---- | ---- | ---- |
| 캐릭터 | 고정된 픽셀아트 | 이미지 파일 교체 가능 (어드민에서 업로드) |
| 장애물 | 고정된 픽셀아트 3종 + 비행 1종 | 이미지 파일 교체 가능, 5종 + 비행 1종 |
| 인트로 화면 | 없음 (바로 시작) | 타이틀 표시 후 시작 |
| 게임 종료 | 장애물 충돌 시 게임오버만 존재 | 게임오버 + 해피엔딩 시나리오 추가 |
| 점수 | 1점 단위 증가 | 50점 단위 증가 |
| 게임오버 UI | 다시하기/순위작성 버튼 | 다시하기 아이콘(↻)만 표시 |
| 사망 연출 | 즉시 정지 | 표정 변화 → 공중 부유 → 낙하 애니메이션 |
| 어드민 페이지 | 없음 | 랭킹 관리 + 이미지 업로드 + 해피엔딩 점수 관리 |
| 배포 | 프론트만 배포 | 프론트 + 백엔드 모두 프로덕션 배포 |
| iframe | 미지원 | iframe 삽입 최적화 |
| 화면 크기 | canvas 600x150 고정 | 모바일 375x208, PC 1920px 폭 |
| 폰트 | 미지정 | EXEPixelPerfect (픽셀 폰트) |

## 2. 화면 설계

### 2.0 인트로 화면

```
┌─────────────────────────────────────────────────┐
│                                                 │
│                                                 │
│             Run Dude Run!                       │
│                                                 │
│                                                 │
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 이미지 준비 완료 후

┌─────────────────────────────────────────────────┐
│                                                 │
│                                                 │
│             Run Dude Run!                       │
│                                                 │
│         PRESS SPACE TO START                    │
│  ─────────────────────────────────────────────── │
│           or tap the screen                     │
└─────────────────────────────────────────────────┘

        ↓ Space/Tap 입력 시

┌─────────────────────────────────────────────────┐
│                                                 │
│                    ↓ (타이틀 아래로 떨어짐)       │
│             Run Dude Run!                       │
│                                                 │
│  🐭→                                            │  ← 쥐가 좌측에서 달려나옴
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 타이틀 "Run Dude Run!"을 화면 중앙에 표시
- 모든 이미지 준비 완료 → "PRESS SPACE TO START" 텍스트 표시
- Space/Tap → 타이틀이 아래로 떨어지는 애니메이션 + 캐릭터(쥐)가 좌측에서 달려나오며 게임 시작

### 2.1 게임 화면

```
┌─────────────────────────────────────────────────┐
│                               00650              │  ← 점수 (우상단, 50점 단위)
│                                                 │
│                                                 │
│                                                 │
│        🐭          🌵       🌵🌵    🦅          │  ← 캐릭터 + 장애물
│  ─────────────────────────────────────────────── │  ← 지면
│                                                 │
└─────────────────────────────────────────────────┘
```

- 게임 화면만 표시
- 점수는 50점 단위로 증가
- 점프 입력: **페이지 전체 영역** 클릭/탭 시 점프 (게임 화면 + 푸터 영역까지)
- 반응형: 모바일 375x208, PC 1920px 폭 기준. 화면 크기에 맞춰 자동 스케일링
- **속도 증가**: 게임이 진행될수록 (점수가 올라갈수록) 화면 스크롤 속도가 점점 빨라져 난이도가 상승합니다 (크롬 다이노 기본 속도 증가 패턴 적용)

### 2.2 Game Over 상태

```
장애물 충돌 시:

┌─────────────────────────────────────────────────┐
│                               00650              │
│                                                 │
│        🐭💀 (표정 변화)                          │  ← 죽은 쥐 스프라이트
│        ↑↑↑ (쥐+장애물 함께 공중에 뜸)             │
│                                                 │
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 공중 부유 후 낙하

┌─────────────────────────────────────────────────┐
│                               00650              │
│                                                 │
│              Game Over                          │
│                                                 │
│                                                 │
│  ─────────────────────────────────────────────── │
│                                                 │
│                  ↻                               │  ← 다시하기 아이콘
│                                                 │
└─────────────────────────────────────────────────┘
```

**사망 애니메이션 흐름**:
1. 장애물 충돌 → 쥐 멈춤 + 표정/포즈 변화 (죽은 쥐 스프라이트)
2. 쥐와 장애물이 함께 잠시 공중에 뜸
3. 아래로 떨어지며 사라짐
4. "Game Over" 표시 + 다시하기 아이콘(↻)

- ↻ 클릭 → "Game Over" 텍스트 사라짐 + 쥐가 왼쪽에서 다시 달려나옴

### 2.3 해피엔딩 화면

```
해피엔딩 트리거 점수 도달 시:

┌─────────────────────────────────────────────────┐
│                               10000              │
│                                                 │
│                                                 │
│                                                 │
│        🐭→                          🐭(흰)      │  ← 주인공이 오른쪽으로, 여친 쥐 대기
│  ─────────────────────────────────────────────── │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 주인공이 여친 쥐에게 도달

┌─────────────────────────────────────────────────┐
│                               10000              │
│                                                 │
│            Love wins all                        │
│                                                 │
│                     🐭♥🐭(흰)                    │  ← 둘 다 눈이 하트
│  ─────────────────────────────────────────────── │
│                                                 │
│                  ↻                               │  ← 다시하기 아이콘
│                                                 │
└─────────────────────────────────────────────────┘
```

**해피엔딩 연출 흐름**:
1. 해피엔딩 트리거 점수 도달 → 장애물이 더 이상 등장하지 않음
2. 화면 오른쪽에 흰색 쥐(여친 쥐)가 가만히 서 있음
3. 주인공이 오른쪽으로 달려감 (기존: 제자리 달리기 → 변경: 주인공이 이동)
4. 주인공이 여친 쥐에게 도달하여 부딪힘
5. 두 쥐의 눈이 하트(♥)로 변함
6. 엔딩 타이틀 "Love wins all" 표시 + 다시하기 아이콘(↻)

**다시하기**:
- ↻ 클릭 → "Love wins all" 텍스트 즉시 사라짐
- 주인공이 왼쪽에서 새로 달려나옴
- 오른쪽의 캐릭터들(여친 쥐 등)은 같은 속도로 오른쪽 화면 밖으로 이동하며 사라짐

### 2.4 모바일 레이아웃

┌──────────────────────┐

│                      │
│    🐭     🌵   🦅   │

## 3. 화면 흐름도

```
┌────────┐  이미지 준비 완료  ┌────────┐  Space/Tap  ┌──────────┐
│ 로딩   │──────────────────▶│ 인트로 │────────────▶│ 게임 중  │
└────────┘                   └────────┘             └──────────┘
                                                        │
                                             ┌──────────┴──────────┐
                                             │                     │
                                          충돌                  클리어
                                             │                     │
                                             ▼                     ▼
                                      ┌───────────┐        ┌────────────┐
                                      │ 게임오버  │        │ 해피엔딩   │
                                      └───────────┘        └────────────┘
                                             │                     │
                                             │  ↻ 클릭             │  ↻ 클릭
                                             │                     │
                                             └──────────┬──────────┘
                                                        │
                                                        ▼
                                                  ┌──────────┐
                                                  │ 게임 중  │
                                                  └──────────┘
```

- **로딩**: 페이지 열림 → 이미지 준비 중. 타이틀만 표시.
- **인트로**: 이미지 준비 완료 → "PRESS SPACE TO START" 표시
- **게임 중**: Space/Tap → 타이틀 떨어지는 연출 + 캐릭터 등장 → 게임 진행
- **게임오버**: 충돌 → 사망 애니메이션 → "Game Over" + ↻ 아이콘
- **해피엔딩**: 트리거 점수 도달 → 여친 쥐 등장 → "Love wins all" + ↻ 아이콘

## 4. 주요 기능

### 4.1 커스텀 스프라이트 (이미지 교체 시스템)

**이미지 포맷**: WebP 권장 (PNG도 허용)

**모든 이미지는 2프레임 루프로 동작** — 달리기, 숙이기, 날아다니는 장애물 모두 2장 교대.

**각 아이템별 권장 이미지 크기**:

| 아이템 | 파일명 | 게임 내 크기 (px) | 권장 원본 크기 (px) | 비고 |
| ------ | ------ | ----------------- | ------------------- | ---- |
| 캐릭터 달리기 | dino-run-1.webp, dino-run-2.webp | 88 x 96 | 176 x 192 (2배) | 2프레임 루프 |
| 캐릭터 숙이기 | dino-duck-1.webp, dino-duck-2.webp | 118 x 60 | 236 x 120 (2배) | 2프레임 루프, 넓고 낮게 |
| 캐릭터 점프 | dino-jump-1.webp, dino-jump-2.webp | 88 x 96 | 176 x 192 (2배) | 2프레임 루프 |
| 캐릭터 사망 | dino-dead-1.webp, dino-dead-2.webp | 88 x 96 | 176 x 192 (2배) | 2프레임 루프 |
| 해피엔딩 쥐 | dino-happy-1.webp, dino-happy-2.webp | 추후 확정 | 추후 확정 | 눈이 하트, 2프레임 루프 |
| 여친 쥐 (대기) | girlfriend-idle-1.webp, girlfriend-idle-2.webp | 추후 확정 | 추후 확정 | 해피엔딩 시 화면 오른쪽 대기 |
| 해피엔딩 여친 쥐 | girlfriend-happy-1.webp, girlfriend-happy-2.webp | 추후 확정 | 추후 확정 | 눈이 하트 |
| 장애물 1 | obstacle-1-1.webp, obstacle-1-2.webp | 추후 확정 | 추후 확정 | 지면 장애물 |

| 장애물 3 | obstacle-3-1.webp, obstacle-3-2.webp | 추후 확정 | 추후 확정 | 지면 장애물 |
| 장애물 4 | obstacle-4-1.webp, obstacle-4-2.webp | 추후 확정 | 추후 확정 | 지면 장애물 |
| 장애물 5 | obstacle-5-1.webp, obstacle-5-2.webp | 추후 확정 | 추후 확정 | 지면 장애물 |
| 날아다니는 장애물 | obstacle-fly-1.webp, obstacle-fly-2.webp | 84 x 60 | 168 x 120 (2배) | 공중 장애물 |
| 지면 타일 | ground.webp | 2400 x 24 | 4800 x 48 (2배) | 가로로 반복 스크롤 |

> **게임 내 크기**는 게임 화면에서 실제로 표시되는 크기입니다. **권장 원본 크기**는 고해상도 화면에서도 선명하게 보이도록 2배 크기로 제작한 원본입니다. 원본 이미지를 전달해 주시면, 게임에서 자동으로 적절한 크기로 축소하여 사용합니다.
> 초기 구현 시 이미지 리소스 없이 컬러 박스 placeholder로 동작합니다.

**이미지 교체 방법**: 어드민 페이지에서 각 이미지를 업로드하여 교체. 업로드 즉시 반영 (프론트엔드 재배포 불필요).

### 4.2 점수 시스템

- **점수 증가**: 50점 단위로 증가 (기존 1점 단위에서 변경)
- **속도 증가**: 크롬 다이노 게임의 속도 증가 패턴을 그대로 적용
- **해피엔딩 트리거**: 일정 점수 도달 시 해피엔딩 진입. 트리거 점수는 어드민에서 변경 가능.

### 4.3 어드민 페이지

별도 페이지에서 랭킹 및 게임 설정을 관리.

```
┌─────────────────────────────────────────────────┐
│              DINO GAME ADMIN                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  [로그인 폼]                                     │
│  비밀번호: [____________]  [로그인]               │
│                                                 │
└─────────────────────────────────────────────────┘

        ↓ 로그인 성공 후

┌─────────────────────────────────────────────────┐
│  DINO GAME ADMIN                    [로그아웃]   │
├─────────────────────────────────────────────────┤
│                                                 │
│  [랭킹 관리] [이미지 관리] [게임 설정]            │
│                                                 │
│  ── 랭킹 관리 ──────────────────────────────     │
│                                                 │
│  RANKING (전체 N건)                              │
│                                                 │
│  #  | 연락처          | 점수 | 날짜       │
│  1  | 010-1234-5678  | 1200 | 03-30     │
│  2  | 010-5678-1234  |  980 | 03-29     │
│  ...                                            │
│                                                 │
│  각 행: [수정] [삭제]                             │
│                                                 │
│  ── 이미지 관리 ────────────────────────────     │
│                                                 │
│  달리는 쥐:      [현재 이미지] [업로드]           │
│  점프 쥐:        [현재 이미지] [업로드]           │
│  죽은 쥐:        [현재 이미지] [업로드]           │
│  해피엔딩 쥐:    [현재 이미지] [업로드]           │
│  여친 쥐:        [현재 이미지] [업로드]           │
│  장애물 1~5:     [현재 이미지] [업로드]           │
│  나는 장애물:    [현재 이미지] [업로드]           │
│  지면 타일:      [현재 이미지] [업로드]           │
│                                                 │
│  ── 게임 설정 ────────────────────────────       │
│                                                 │
│  해피엔딩 트리거 점수: [10000]  [저장]            │
│                                                 │
└─────────────────────────────────────────────────┘
```

**기본 기능**:
- 비밀번호 기반 로그인
- 로그인 시도 횟수 제한 (5회 실패 시 일시 차단)
- 일정 시간 동안 조작이 없으면 자동 로그아웃
- 연락처는 목록에서 일부가 가려진 상태로 표시 (예: `010-****-5678`). 전체 번호는 개별 항목 클릭 시에만 확인 가능
- 전체 랭킹 조회, 수정, 삭제 가능
- HTTPS 필수 적용

**이미지 업로드 관리**:
- 각 캐릭터/장애물 이미지를 어드민에서 업로드하여 교체
- 업로드 시 픽셀 크기 검증 (지정 크기와 불일치 시 업로드 거부)
- WebP/PNG 포맷만 허용
- 업로드 즉시 반영 (프론트엔드 재배포 불필요)

**해피엔딩 점수 관리**:
- 해피엔딩 트리거 점수를 어드민에서 조회/변경 가능
- 변경 시 즉시 반영

### 4.4 스토어 페이지 삽입

이 게임은 독립된 웹페이지로 만들어지며, 온라인 스토어 페이지 안에 **작은 창(iframe)** 형태로 끼워 넣을 수 있습니다. 스토어 페이지를 방문한 고객은 별도의 페이지 이동 없이, 해당 위치에서 바로 게임을 플레이할 수 있습니다.

- 게임 영역을 클릭하면 자동으로 게임에 포커스가 맞춰져 바로 조작할 수 있습니다
- 허용된 스토어 페이지에서만 게임이 동작하도록 보안 설정이 적용됩니다

## 5. FAQ

**왜 상위 500건만 보관하나요?**
- 프로모션 기간 동안 수천 건 이상의 데이터가 쌓일 수 있으나, 경품 지급 등 실질적으로 필요한 데이터는 상위권에 한정됩니다. 불필요하게 많은 데이터를 보관하지 않고 상위 500건만 유지하여 관리를 간소화합니다.

**해피엔딩 트리거 점수는 어떻게 결정하나요?**
- 어드민 페이지에서 자유롭게 변경할 수 있습니다. 프로모션 상황에 따라 난이도를 조절할 수 있습니다.

## 6. 유저 플로우

```
페이지 열림 → 타이틀 "Run Dude Run!" 표시 + 이미지 준비
  │
  ▼
이미지 준비 완료 → "PRESS SPACE TO START" 표시
  │
  ▼
Space/Tap → 타이틀 떨어지는 연출 + 캐릭터 등장 → 게임 시작
  │
  ▼
게임 진행 (달리기, 점프, 숙이기 / 점수 50점 단위 증가)
  │
  ├─ 장애물 충돌 → 사망 애니메이션 (표정 변화 → 공중 부유 → 낙하)
  │   │
  │   ▼
  │   "Game Over" + ↻ 아이콘
  │   │
  │   └─ ↻ 클릭 → 게임 재시작
  │
  └─ 해피엔딩 트리거 점수 도달 → 장애물 중단
       │
       ▼
       여친 쥐 등장 → 주인공이 달려감 → 만남
       │
       ▼
       두 쥐 눈이 하트 → "Love wins all" + ↻ 아이콘
       │
       └─ ↻ 클릭 → 게임 재시작
```

## 8. 구현 마일스톤

| Step | 목표 | 주요 산출물 |
| ---- | ---- | ---------- |
| Step 1 | 인트로 화면 + 이미지 교체 시스템 | 인트로 연출, 이미지 로딩, placeholder 동작 |
| Step 2 | 게임오버 UX + 사망 연출 | 사망 애니메이션, 다시하기 아이콘, 점수 50점 단위 |
| Step 3 | 해피엔딩 시나리오 | 트리거 로직, 여친 쥐 등장, 엔딩 연출 |
| Step 4 | 백엔드 + 어드민 페이지 | 랭킹 관리, 이미지 업로드, 해피엔딩 점수 관리 |
| Step 5 | iframe 최적화 + 배포 | iframe 보안, 프로덕션 배포, 테스트 |

> 각 Step 완료 후 리뷰 진행, 확인 후 다음 Step 착수.
