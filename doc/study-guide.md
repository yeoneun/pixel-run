# 풀스택 바이브코딩 스터디 가이드

> 출처: [Notion - 풀스택 바이브코딩 스터디](https://www.notion.so/cupist-inc/313f48194caa8084a86ac3be2f341c65)

## 스터디 소개

AI에게 물어보면서 서버를 직접 만들고, DB를 통해 **CRUD가 어떤 과정을 통해 동작하는지** 이해하는 것이 목표.
완성도보다 **과정에서 알게 된 것**을 공유하는 스터디.

## 운영 방식

- **주기**: 격주 1회 모임 (총 4~5회, 약 2~3개월) 각자 약 10분 공유

## 기술 스택

| 항목 | 스택 |
|------|------|
| 인프라 | Docker Compose (로컬) |
| DB | PostgreSQL 16.11 |
| 백엔드 | NestJS (Raw SQL → ORM 순서로 진행) |
| 프론트 | 자유 (React Native, Next.js, Electron 등) |

## 공유 방식 (1인 약 10분)

1. 동작 화면 공유 (동영상 or 스크린샷)
2. 이번 미션에서 새로 알게 된 것 1~2가지
3. 막혔던 것 or 삽질한 것

---

## 1회차: 환경 셋업 & Create / Read

### 목표

환경을 구성하고, 데이터를 DB에 넣고 꺼내본다. Raw SQL로 직접 쿼리를 작성한다.

### 할 일

- Docker Compose로 PostgreSQL 16.11 컨테이너 띄우기
- DB에 접속해서 쿼리가 실행되는지 확인하기
- NestJS 프로젝트에 DB 연결하기 (ORM 없이 `pg` 라이브러리 사용)
- SQL로 테이블 1개 만들기
- POST API → INSERT 쿼리로 데이터 저장
- GET API → SELECT 쿼리로 데이터 조회
- RN에서 등록/조회 화면 연결

### 이해해볼 것

- HTTP 요청이 NestJS를 거쳐 DB까지 어떤 경로로 도달하는지
- DB에서 돌아온 데이터가 RN 화면까지 어떻게 오는지

### 완료 기준

- [ ] `docker compose up`으로 PostgreSQL이 뜬다
- [ ] NestJS 서버가 뜨고 DB 연결이 된다
- [ ] POST API로 데이터가 DB에 들어간다
- [ ] GET API로 데이터를 조회할 수 있다
- [ ] RN 앱에서 등록/조회가 동작한다

---

## 2회차: Update & Delete

### 목표

CRUD를 완성한다. 각 SQL 쿼리가 데이터를 어떻게 변경하고 삭제하는지 이해한다.

### 할 일

- PATCH API → UPDATE 쿼리로 데이터 수정
- DELETE API → DELETE 쿼리로 데이터 삭제
- GET :id API → 단건 조회
- RN에서 수정/삭제 기능 연동
- (선택) Swagger 설정하기

### 이해해볼 것

- UPDATE와 DELETE에서 WHERE 절이 왜 중요한지
- CRUD 4개 API가 각각 어떤 SQL을 실행하는지 전체 그림 그려보기

### 완료 기준

- [ ] 데이터를 수정할 수 있다
- [ ] 데이터를 삭제할 수 있다
- [ ] RN 앱에서 CRUD가 전부 동작한다
- [ ] 각 API가 어떤 SQL을 실행하는지 설명할 수 있다

---

## 3회차: ORM 도입 & 테스트 코드

### 목표

1~2회차에서 Raw SQL로 만든 CRUD를 ORM으로 다시 구현한다.
ORM이 SQL을 어떻게 대신 만들어주는지 비교하고, 테스트 코드를 작성해본다.

### 할 일

- ORM 도입 (Prisma)
- 기존 테이블에 맞는 모델/스키마 정의
- 1~2회차에서 만든 CRUD를 ORM으로 다시 구현
- Raw SQL 코드와 ORM 코드 비교
- CRUD API에 대한 테스트 코드 작성

### 이해해볼 것

- ORM이 내부적으로 어떤 SQL을 만들어주는지
- Raw SQL로 직접 짰을 때와 ORM을 썼을 때 뭐가 다른지
- 마이그레이션이 뭐고 왜 필요한지
- 테스트 코드가 왜 필요한지
- API를 테스트하는 방법

### 완료 기준

- [ ] ORM으로 CRUD가 동작한다
- [ ] Raw SQL과 ORM 코드를 비교해서 설명할 수 있다
- [ ] ORM이 생성하는 SQL을 확인해봤다
- [ ] CRUD API에 대한 테스트 코드가 있고 통과한다

## 4회차: 회원가입/로그인 & 데모

### 목표

인증 기능을 구현한다. 회원가입/로그인이 백엔드에서 어떻게 동작하는지 이해한다.

### 할 일

- 유저 테이블 만들기
- 회원가입 API (비밀번호 암호화)
- 로그인 API (JWT 토큰 발급)
- 인증이 필요한 API 만들어보기
- RN에서 로그인/회원가입 화면 연결

### 이해해볼 것

- JWT 토큰이 어떻게 동작하는지
- 인증된 요청과 인증 안 된 요청의 차이

### 완료 기준

- [ ] 회원가입/로그인이 동작한다
- [ ] 인증이 필요한 API가 토큰 없이 호출하면 거부된다
